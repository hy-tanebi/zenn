## はじめに

自社のコーポレートサイトで、マウスホバーに追従する「水の屈折」のようなエフェクトを実装しました。

**実際のサイト**: [tanebi-net.com](https://tanebi-net.com)

![ホバーエフェクトのデモ](/images/hover-fluid-demo.gif)
_マウスを動かすと、画面全体が水面のように歪む_

この記事では、react-three-fiber + カスタムGLSLシェーダーを使ったこのエフェクトの仕組みを、実装コード付きで解説します。

### この記事で学べること

- FBO（FrameBuffer Object）を使った画面キャプチャと再合成
- FBMノイズによる自然な「水面ゆらぎ」の作り方
- マウス追従の「とろみ」を生む指数平滑化
- 60fpsを維持するためのパフォーマンス最適化

---

## 仕組みの全体像

このエフェクトは、いわば **「画面のスクリーンショットを毎フレーム撮影し、それをシェーダーで歪ませて表示する」** という仕組みです。

```
通常の3Dシーン描画
        ↓
  FBO（オフスクリーンバッファ）に記録
        ↓
  全画面メッシュのシェーダーで歪みを計算
        ↓
  歪んだ画面を表示
```

ポイントは、歪み処理を行うメッシュ自身はFBO撮影時に非表示にすること。そうしないと「歪みの歪み」が無限に重なってしまいます。

---

## セットアップ: react-three-fiber + FBO

### FBOの生成

dreiの `useFBO` で画面キャプチャ用のバッファを作ります。

```tsx
import { useFBO } from "@react-three/drei";

const fboWidth = typeof window !== "undefined" ? window.innerWidth : 1024;
const fboHeight = typeof window !== "undefined" ? window.innerHeight : 1024;
const fbo = useFBO(fboWidth, fboHeight, { samples: 0 });

// 色空間を揃える（色ズレ防止）
useEffect(() => {
  fbo.texture.colorSpace = THREE.LinearSRGBColorSpace;
}, [fbo]);
```

`samples: 0` でアンチエイリアスを無効にし、パフォーマンスを確保しています。

### カスタムシェーダーマテリアル

dreiの `shaderMaterial` でGLSLシェーダーを登録します。

```tsx
import { shaderMaterial } from "@react-three/drei";
import { extend } from "@react-three/fiber";

export const HoverFluidMaterial = shaderMaterial(
  {
    uScene: null, // FBOテクスチャ
    uResolution: new THREE.Vector2(1, 1), // 画面解像度
    uMouse: new THREE.Vector2(0.5, 0.5), // マウス位置（0〜1）
    uVel: new THREE.Vector2(0, 0), // マウス速度
    uTime: 0, // 経過時間
    uIntensity: 0, // エフェクト強度
    uRadius: 0.08, // 影響半径
    uFalloff: 0.12, // 減衰幅
    uDispAmp: 0.05, // 歪み振幅
    uNoiseAmp: 0.45, // ノイズ強度
    uBaseAmp: 0.015, // 常時ゆらぎ振幅
    uBaseScale: 1.2, // ゆらぎスケール
    uChromAb: 0.0, // 色収差
  },
  vertexShader,
  fragmentShader,
);

extend({ HoverFluidMaterial });
```

### 全画面メッシュの配置

`PlaneGeometry(2, 2)` で画面全体を覆うメッシュを作り、最前面に描画します。

```tsx
<mesh
  ref={fluidRef}
  renderOrder={10000} // 全オブジェクトの最前面
  frustumCulled={false} // 画面外でも描画（全画面なので必須）
  raycast={() => null} // マウスピッキング対象外
>
  <planeGeometry args={[2, 2, 1, 1]} />
  <hoverFluidMaterial
    ref={hoverMatRef}
    transparent={true}
    depthTest={false}
    depthWrite={false}
    blending={THREE.NoBlending}
    toneMapped={false}
    uTexture={fbo.texture}
  />
</mesh>
```

頂点シェーダーは最小限です。投影行列を無視して、NDC座標に直接出力します。

```glsl
// hover-fluid-vertex.glsl
varying vec2 vUv;
void main() {
    vUv = uv;
    gl_Position = vec4(position.xy, 0.0, 1.0);
}
```

### FBOへのシーン描画

`useFrame` 内で毎フレーム、通常のシーンをFBOに描画します。

```tsx
useFrame((state, delta) => {
  if (fluidRef.current) {
    // 1. 歪みメッシュを一時非表示
    fluidRef.current.visible = false;

    // 2. シーンをFBOに描画
    state.gl.setRenderTarget(fbo);
    state.gl.clear();
    state.gl.render(state.scene, state.camera);
    state.gl.setRenderTarget(null);

    // 3. 歪みメッシュを再表示
    fluidRef.current.visible = true;
  }
});
```

これで「歪む前の画面」がFBOテクスチャとしてシェーダーに渡ります。

## シェーダーの実装: 水面を作る

ここからが核心です。フラグメントシェーダーで「どうやって水面のような歪みを作るか」を解説します。

### ノイズ関数: 自然な凹凸を生む

水面の歪みには、規則的なパターンではなく**自然で不規則な起伏**が必要です。これを作るのがFBM（Fractal Brownian Motion）ノイズです。

```glsl
// 疑似ランダムハッシュ
float hash(vec2 p) {
  p = fract(p * vec2(123.34, 345.45));
  p += dot(p, p + 34.345);
  return fract(p.x * p.y);
}

// 2Dバリューノイズ
float noise(vec2 p) {
  vec2 i = floor(p), f = fract(p);
  float a = hash(i);
  float b = hash(i + vec2(1.0, 0.0));
  float c = hash(i + vec2(0.0, 1.0));
  float d = hash(i + vec2(1.0, 1.0));
  vec2 u = f * f * (3.0 - 2.0 * f);  // Hermite補間
  return mix(mix(a, b, u.x), mix(c, d, u.x), u.y);
}

// FBM: ノイズを重ね合わせて複雑な起伏を生成
float fbm(vec2 p) {
  float v = 0.0;
  float a = 0.5;
  for(int i = 0; i < 3; i++) {
    v += a * noise(p);
    p *= 2.0;   // 周波数を倍に
    a *= 0.5;   // 振幅を半減
  }
  return v;
}
```

FBMは「大きなうねり + 中くらいの波 + 細かいさざ波」を合成することで、自然な水面を表現します。ループ回数3は、見た目と性能のバランスで決めた値です（後述のパフォーマンスセクション参照）。

### グラデーション関数: ノイズから「歪み方向」を得る

ノイズ値そのものはスカラー（数値1つ）なので、UV座標をどの方向にずらすかは分かりません。そこで**数値微分**でグラデーション（勾配）を計算し、歪みのベクトルを得ます。

```glsl
vec2 grad(vec2 p) {
  float e = 0.0015;
  float n1 = fbm(p + vec2(e, 0.0));
  float n2 = fbm(p + vec2(-e, 0.0));
  float n3 = fbm(p + vec2(0.0, e));
  float n4 = fbm(p + vec2(0.0, -e));
  return vec2(n1 - n2, n3 - n4);
}
```

微小なオフセット `e` でノイズの差分を取ることで、「この位置でノイズがどの方向に傾いているか」＝歪みの方向が分かります。

### ベースのゆらぎ: 常に揺れる水面

マウスが動いていなくても画面全体がわずかに揺れる「水面感」を作ります。

```glsl
vec2 bp = uv * vec2(uBaseScale * aspect, uBaseScale)
       + vec2(uTime * 0.018, -uTime * 0.016);
vec2 baseNrm = grad(bp);
vec2 baseDisp = baseNrm * 0.016;
```

`uTime` を加算することで、ノイズのサンプリング位置が時間とともにずれ、常に流れ続ける水面になります。振幅 `0.016` は「気づくか気づかないか」程度の微弱な値です。

### マウス周辺の歪み: 水に指を入れた感覚

ここがエフェクトの主役です。マウスの位置と速度に応じて、4つの要素を組み合わせます。

#### 楕円マスク

マウスの速度方向に伸びる楕円形の影響範囲を作ります。

```glsl
vec2 vel = uVel;
float vlen = length(vel);
vec2 dir = normalize(vel + 1e-6);
vec2 rel = uv - uMouse;

// 速度方向と垂直方向の成分に分解
float along = dot(rel, dir);
float perp = dot(rel, vec2(-dir.y, dir.x));

// 速度が高いほど楕円が細長くなる
float aniso = mix(1.0, 1.45, clamp(vlen * 3.5, 0.0, 1.0));
float rEll = length(vec2(along / aniso, perp));

float mask = 1.0 - smoothstep(uRadius, uRadius + uFalloff, rEll);
mask *= clamp(uIntensity, 0.0, 1.0);
```

マウスをゆっくり動かすと円形、素早く動かすと速度方向に伸びた楕円になります。

#### 膨らみ・尾部・とろみノイズ

```glsl
// 膨らみ: マウスの進行方向前方が膨らむ
vec2 bulgeC = uMouse + dir * (0.025 * vlen);
float bulge = 1.0 - smoothstep(0.0, uRadius, length(uv - bulgeC));
bulge *= mask;

// 尾部: 素早く動くと後方に「引きずり」が残る
vec2 tailC = uMouse - dir * (0.16 * vlen);
float tail = 1.0 - smoothstep(
  uRadius * 1.7, uRadius * 1.7 + uFalloff * 1.3, length(uv - tailC)
);
float ghost = tail * smoothstep(0.28, 0.6, vlen) * 0.55;

// とろみノイズ: 渦巻くような流動感を追加
vec2 nUv = uv * vec2(6.0 * aspect, 6.0)
         + vec2(uTime * 0.55, -uTime * 0.48) + vel * 1.4;
float n1 = fbm(nUv);
float n2 = fbm(nUv + vec2(19.3, -11.7));
vec2 flow = (vec2(n1, n2) - 0.5) * 2.0;
```

### 最終合成: すべてを混ぜる

各要素を方向ベクトルとして合成し、UV座標のオフセット量を決定します。

```glsl
vec2 ndir = normalize(rel + 1e-6);       // マウスからの方向
vec2 tangent = vec2(-ndir.y, ndir.x);    // 接線方向

vec2 disp = ndir * (0.70 * bulge) +       // 放射状の膨らみ
  tangent * (0.38 * bulge) +              // 接線方向の渦
  flow * (uNoiseAmp * mask) +             // 流動ノイズ
  (-dir) * (0.30 * ghost);               // 後方の引きずり

// ベースゆらぎ + マウス歪み を合算
vec2 dispCombined = baseDisp * uBaseAmp + disp * uDispAmp;
```

最後に、計算した歪みベクトルでFBOテクスチャのサンプリング位置をずらします。

```glsl
vec2 uvShifted = clamp(uv + dispCombined, vec2(0.001), vec2(0.999));
gl_FragColor = texture2D(uScene, uvShifted);
```

たったこれだけで、画面が水面のように歪みます。

## マウス追従の滑らかさ: 指数平滑化

生のマウス座標をそのまま使うと、カクカクした動きになります。「水がとろっと追いかけてくる」感覚を出すために、**指数平滑化**（Exponential Smoothing）を適用します。

```tsx
useFrame((state, delta) => {
  // 位置の平滑化
  const kMouse = 1 - Math.exp(-delta * 12.0);
  mouseFiltered.current.lerp(mouseRaw.current, kMouse);

  // 速度の計算
  const velNow = new THREE.Vector2()
    .subVectors(mouseFiltered.current, lastMouseFiltered.current)
    .multiplyScalar(1 / Math.max(1e-6, delta));

  // 速度の平滑化
  const kVel = 1 - Math.exp(-delta * 20.0);
  velSmoothed.current.lerp(velNow, kVel);

  lastMouseFiltered.current.copy(mouseFiltered.current);
});
```

### なぜ `1 - exp(-delta * λ)` なのか

この式は**フレームレート非依存の平滑化**を実現します。

```
k = 1 - e^(-δ × λ)
```

- `δ`（delta）: 前フレームからの経過時間
- `λ`: 平滑化の強さ（大きいほど速く追従）

60fpsなら `δ ≈ 0.0167`、30fpsなら `δ ≈ 0.033` ですが、`k` の計算にデルタタイムが含まれているため、**どのフレームレートでも同じ速度で追従します**。

単純な `lerp(raw, 0.1)` だと、フレームレートが倍になると追従速度も倍になってしまいます。

| パラメータ | λ値  | 60fpsでのk | 追従の速さ               |
| ---------- | ---- | ---------- | ------------------------ |
| マウス位置 | 12.0 | ≈ 0.18     | 適度な「とろみ」         |
| マウス速度 | 20.0 | ≈ 0.28     | やや速め（歪みの反応用） |

### エフェクト強度の減衰

マウスが止まったらエフェクトが自然にフェードアウトする仕組みも入れています。

```tsx
// マウスが動いたら 1.0 にセット
hoverStrength.current = 1;

// 毎フレーム減衰（約0.6秒で消滅）
hoverStrength.current = Math.max(0, hoverStrength.current - delta * 1.6);

// ユニフォームへの反映も指数平滑
const kInt = 1 - Math.exp(-delta * 6.0);
u.uIntensity.value += (hoverStrength.current - u.uIntensity.value) * kInt;
```

## パフォーマンス最適化

3Dエフェクトは油断するとすぐに重くなります。実装した最適化を紹介します。

### FBO更新の条件付き実行

FBOへのシーン描画は重い処理です。ホバー中だけ毎フレーム更新し、アイドル時は3フレームに1回に抑えます。

```tsx
frameCountRef.current++;
const isHoverActive = hoverStrength.current > 0.1;
const shouldUpdateFbo =
  isHoverActive || frameCountRef.current - lastFboUpdateRef.current >= 3;

if (fluidRef.current && shouldUpdateFbo) {
  lastFboUpdateRef.current = frameCountRef.current;
  // ... FBO描画処理
}
```

アイドル時は**約66%のGPU負荷を削減**できます。

### FBMループ回数の削減

FBMのループ回数は、見た目への影響が小さい範囲で削減しました。

```glsl
// 変更前: 5回ループ
for(int i = 0; i < 5; i++) { ... }

// 変更後: 3回ループ（約40%高速化）
for(int i = 0; i < 3; i++) { ... }
```

5回から3回に減らしても、人間の目では違いがほとんど分かりません。フラグメントシェーダーは全ピクセルで実行されるため、この削減の効果は大きいです。

### 色収差オフ時の最適化

色収差（Chromatic Aberration）を使わない場合、テクスチャサンプリングを3回から1回に減らします。

```glsl
if(abs(uChromAb) < 0.001) {
  // 1回サンプリング（3倍高速）
  vec2 uvSingle = clamp(uv + dispCombined, vec2(0.001), vec2(0.999));
  col = texture2D(uScene, uvSingle);
} else {
  // R, G, B 別々にサンプリング（色収差効果）
  vec2 uvR = clamp(uv + dispCombined * (1.00 + uChromAb), ...);
  vec2 uvG = clamp(uv + dispCombined * (1.00), ...);
  vec2 uvB = clamp(uv + dispCombined * (1.00 - uChromAb), ...);
  col.r = texture2D(uScene, uvR).r;
  col.g = texture2D(uScene, uvG).g;
  col.b = texture2D(uScene, uvB).b;
}
```

### モバイルでの無効化

モバイル端末ではFBO処理全体をスキップします。

```tsx
const isMobileFrame = state.size.width < 768;

if (fluidRef.current && isMobileFrame) {
  fluidRef.current.visible = false;
}
```

モバイルのGPUでFBO合成を行うとバッテリー消費が激しく、発熱の原因にもなります。エフェクトの有無よりもユーザー体験を優先しました。

## パラメータチューニング

各ユニフォーム値を調整することで、エフェクトの印象を大きく変えられます。

| パラメータ   | 値    | 効果                                         |
| ------------ | ----- | -------------------------------------------- |
| `uRadius`    | 0.095 | 影響範囲の半径。大きくするとぼんやり広がる   |
| `uFalloff`   | 0.165 | エッジの柔らかさ。大きいほどソフト           |
| `uDispAmp`   | 0.045 | 歪みの強さ。大きすぎると不自然に             |
| `uNoiseAmp`  | 0.38  | 流動ノイズの強さ。大きいとざらざらした質感に |
| `uBaseAmp`   | 0.013 | 常時ゆらぎの強さ。微弱でOK                   |
| `uBaseScale` | 1.18  | ゆらぎの粗さ。大きいと細かい波に             |
| `uChromAb`   | 0.0   | 色収差。0.02〜0.05で虹色の分散効果           |

### デバッグマスクの活用

`uShowMask` を `1` にすると、各マスク領域が色分けで可視化されます。

```glsl
if(uShowMask == 1) {
  vec3 dbg = mix(vec3(0.0), vec3(0.2, 0.7, 1.0), heroMask);  // 青: ヒーロー領域
  dbg = mix(dbg, vec3(1.0, 0.2, 0.2), mask);                  // 赤: ポインター主マスク
  dbg = mix(dbg, vec3(1.0, 1.0, 0.3), ghost);                 // 黄: 尾部
  gl_FragColor = vec4(dbg, 1.0);
}
```

パラメータ調整時はこのデバッグモードを有効にすると、「どこにどんな影響範囲があるか」を一目で確認できます。

## まとめ

この記事で紹介した水の屈折エフェクトは、以下の要素の組み合わせです。

1. **FBO** で画面をキャプチャし、シェーダーに渡す
2. **FBMノイズのグラデーション** で自然な水面のゆらぎを生成
3. **楕円マスク + 膨らみ + 尾部 + 流動ノイズ** でマウス周辺の動的な歪みを作る
4. **指数平滑化** で「とろみ」のある滑らかな追従を実現
5. **条件付きFBO更新 + FBM削減 + モバイル無効化** でパフォーマンスを確保
